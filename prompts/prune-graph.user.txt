## 統合（merge）の判断基準 — 該当するものは**必ず**統合
- 同じ概念の異なる表記（例: "ML" と "機械学習"、"React.js" と "React"）
- 表記ゆれ（例: "データベース" と "DB"、"サーバー" と "サーバ"）
- 英語と日本語の同義語（例: "認証" と "Authentication"）
- 略語と正式名称（例: "AI" と "人工知能"）
- 上位概念と具体例で、具体例側が情報不足の場合（例: "技術" は "Web技術" に統合）
- 同じ分野で非常に近い概念（例: "深層学習" と "ディープラーニング"、"自然言語処理" と "NLP"）
- 同じ技術やツールの異なる側面（例: "Pythonライブラリ" と "Pythonパッケージ"）
- 統合先は、より具体的・認知度の高い方を選ぶ

## 削除（remove）の判断基準 — 該当するものは**必ず**削除
- 一般的すぎて意味のないキーワード（例: "情報", "技術", "システム", "データ", "処理", "方法", "開発", "研究", "最新", "動向"）
- 文の断片や不完全なキーワード
- 広告・スポンサー由来のキーワード
- 調査回数が1回のみで接続が0の孤立キーワード
- 意味が不明確なキーワード
- 固有名詞でない一般的な動詞・形容詞
- 他のキーワードと内容が重複するが統合先がないもの
